消息认证码

发表于 2024-11-19 更新于 2024-11-21

对应教材第四章Message Authentication Codes

消息完整性

保密性vs完整性

密码学的基本目标之一是让通信双方安全通信。我们已经讲过如何实现保密性，即通过加密明文防止窃听者能在开放信道上获取有关消息的任何信息。

但是，对于一些能够注入信息或者纂改信息的主动攻击者，实现消息认证（完整性）就更加重要。比如发送的消息是否是用户本人发出，消息在传输过程中是否被纂改过。这里要注意的是，常规的错误纠正技术只针对与传输过程中的随机错误，无法防御恶意攻击者对内容的精确修改。

以下是一个实际的例子。购物网站会将一些信息存储在Cookie中，包含了一些商品的价格折扣等等，而Cookie由用户进行保存。显然商家不希望用户对Cookie进行修改，而显然Cookie的内容对用户来说是明文，因为用户必须知道这些信息，所以这里的问题就是一个完整性问题。

所以这章的目标是用密码学技术实现消息的完整性(integrity)，以检测伪造的消息或在未受保护的通信信道上发送的被篡改的消息。需要注意的是，我们无法完全防止消息注入或篡改，因为这种攻击只能在物理层面加以防御。相反，我们的目标是确保任何此类行为都能被honest parties检测到。

加密vs消息认证

保密性和信息完整性是截然不同的目标。我们需要明确的是：除非加密是专门为完整性设计的，否则不会提供任何完整性保障。

可能很容易产生一个误解，既然密文完全隐藏了消息的内容，那么攻击者似乎不可能以任何有意义的方式篡改加密的消息。尽管这种推理直观上似乎合理，但它实际上是完全错误的。下面将举例说明这一点。

流密码加密

假设一种异或处理的流密码，只需要在密文中任意翻转一位bit，那么解密后的明文就会也翻转一个bit，假如攻击方对加密密文由一定认识，很有可能翻转敏感重要的数字位来造成攻击。

块密码加密

块密码一般来说单个位的比特变化会导致后续的变化，从而被发现。但是，如果是使用ECB模式时，翻转密文中第i个块的一个位只会影响明文中的第i个块——其他块保持不变。这样就可以构成有效的攻击。

最后需要注意的是，我们迄今为止讨论的所有加密方案都有一个共同的特性：任何长度合适的字符串都是有效的密文，因此都对应于某个有效的明文。基于这一特性，即使攻击者完全不知道密文对应的明文内容，也可以通过发送一条长度正确的任意字符串来“伪造”一方的消息。这使得攻击者能够轻松冒充通信方发送消息。

在消息完整性的语境中，即便是这种简单形式的攻击也应当被彻底排除。

消息认证码的定义(MACs)

现在我们已经了解到，加密通常不能解决消息完整性的问题，所以我们引入消息认证码(Message Authentication Code, MAC)来让通信双方有能力判断消息是否被纂改。

想要实现这一点，通信双方必须要共享某个攻击者不知道的消息。这里我们继续讨论对称密钥的场景，即通信双方共享一个密钥。

MAC有两个典型应用场景：

  • 确保双方通信完整性：比如用户和银行之间的通信。
  • 在时间跨度内保护单个用户的数据完整性，比如之前提到过的Cookie，或者用户硬盘。

MAC的定义

首先先明确MAC是什么以及怎么使用。

首先，通信双方要生成一个共享密钥$key$。当一方想向另一方发送消息$m$时，会根据消息和密钥生成一个认证标签$t$，这个标签由标签生成算法Mac完成。接收方收到密文和标签后，会对其进行验证，验证消息在中途有没有收到纂改，验证由验证算法Vrfy完成。

形式化定义如下：

DEFINITION 4.1
一个消息验证码由三个概率多项式算法组成(Gen,Mac,Vrfy)。

  1. Gen接受$1^n$作为输入，输出$k,|k|\ge n$。
  2. Mac接受$k,m$作为输入，输出$t$。因为这可能是一个概率算法，所以写作$t\leftarrow Mac_k(m)$
  3. Vrfy接受$k,m,t$作为输入，输出$1$或$0$。

$Vrfy(m,Mac_k(m),t)=1$

如果存在一个函数$l$，使得算法$Mac_k$仅对长度位$l(n)$的消息定义，那么该方案为固定长度MAC。

与对称密钥加密类似，一般Gen算法均匀选取key，所以一般省去对Gen的描述。

标准认证(Canonical Verification)：对于确定性消息验证码，即MAC是确定性算法，只需再计算一遍tag比较是否相同即可验证。这里说明每条消息对应唯一的标签。

MAC的安全性

我们现在来定义一个MAC的安全性概念。该定义的核心思想是：任何有效的攻击者都无法为任何“新”消息生成有效的标签，其中“新”消息指的是此前未被通信方发送（并认证）的消息。

按照惯例，我们只考虑概率多项式时间的攻击者。在消息认证的场景中，攻击者观察通信方之间的交互可能能够看到所有发送的消息及其对应的标签。此外，攻击者可能通过直接或间接方式影响这些消息的内容（例如，攻击者的外部行为影响通信方发送的消息）。

攻击者能力建模：我们允许攻击者请求任意合法消息的标签。即为供给者提供一个$Mac_k()$的访问权限。

攻击者的目标：生成一个新消息和一个对应的正确标签。如果在多项式时间内成功生成，则称破坏了方案的安全性。

如果消息认证码在以上情形下无法被攻破，那么称其为在自适应选择消息攻击下具有存在性不可伪造性（existentially unforgeable under an adaptive chosen-message attack, EUF-CMA）。该安全性要求确保，即便攻击者能够获取任意消息的标签，也无法伪造出新消息及其有效标签。

实验定义：

  1. 生成key
  2. 攻击者获得$Mac_k()$的访问权限，可以多次查询，最终攻击者输出一对$(m,t)$。
  3. 当且仅当输出的m没有被查询过，且$Vrfy(m,t)=1$时，攻击成功。

DEFINIEION 4.2
一个消息认证码是existentially unforgeable under an adaptive chosen-message attack，当且仅当，对于任意的多项式算法攻击者，都存在一个negl使：
$Pr[Mac-forge_{\mathcal{A},\prod}(n)=1]\leq negl(n)$

这里的对安全性的定义看起来很严格：一方面攻击者可以自由选择信息并获得标签，另一方面只要伪造出任意的消息和标签就能成功攻破。这样的定义是为了保证方案的通用性和实用性。

重放攻击：到目前位置没有考虑过以下情形，攻击者简单发送一条已经认证过的消息和其对应的有效标签。

MAC本身对重放攻击是无法防御的，因为其验证过程是无状态的。因此，会在更高层的应用中处理重放攻击，比如使用序列号和使用时间戳来防御。

强不可伪造性Strong unforgeability：
在标准定义中对安全性的保证是不能对新消息生成正确的标签，但是没有限制对已经认证过的消息生成一个新的有效标签。现在我们定义一个强不可伪造性实验Mac-sforge，要求目标是攻击者无法为已认证的消息生成有效的新标签。从而定义了一种更强的安全性。

DEFINIEION 4.3
一个消息认证码是strongly secure，当且仅当对于任意的多项式算法攻击者，都存在一个negl使：
$Pr[Mac-sforge_{\mathcal{A},\prod}(n)=1]\leq negl(n)$

PROPOSITION 4.4
如果一个消息认证码方案在标准验证下是安全的，那么这是strongly secure。

验证查询(Verification queries)：定义4.2和定义4.3给了攻击者访问MAC的权限，即可以获得消息对应的标签。但是攻击者也可能获得验证的权限，即可以发送$(m,t)$给接收方并得知$Vrfy_k(m,t)$的结果。
攻击者引入验证查询后，对于使用标准验证的方案，不影响对安全性的定义，仍然是强安全的。但是对于一般的MAC方案，可能会影响定义的结果。这很好理解，因为如果是标准验证，标签是唯一的。
下面讨论的方案大多数都是强安全的，所以我们下面会忽略对验证查询的部分。

潜在的时间攻击：
在现实生活中，攻击者不仅可以得知自己发送的$(m,t)$是否被接受，还能知道接收方判断所花费的时间，这也可能成为一种攻击的手段。
最简单的例子：假设接收方验证算法是逐字符比较，就可以知道正确的部分有多长。
所以验证算法应该使用和时间无关的算法。

图灵机

发表于 2024-11-18

什么是图灵机

直觉上

图灵机长这样

有一条无限长的纸带，纸带里有一些方格，方格中的符号来自一个有限字母表。

图灵机可以根据自己当前的状态和读写头指向的纸带符号，改变自己的状态，重写读写头指向的符号，然后将读写头移动一个方格（可以向左也可以向右）。

值得注意的是，图灵机和一般的计算机都有等价的表达能力，而因为图灵机有无限内存（纸带无限长），图灵机甚至比物理上的计算机更加强大。

形式化定义

一个图灵机是一个七元组 $M=(Q,\sum,\Gamma,\delta,q_0,B,F)$

  • $Q$是一个有限的状态集合。
  • $\sum$是一个输入字母表。
  • $\Gamma$是一个纸带字母表，$\Gamma \subseteq \sum$。
  • $\delta$是转移函数。
    • 接受两个参数，一个状态$q\in Q$和一个纸带符号$Z\in\Gamma$
    • $\delta(q,Z)$要么没定义，要么为一个三元组$(p,Y,D)$，分别是状态，纸带符号，转换头前进方向（L或R）。
  • $q_0$是起始状态。
  • $B$是一个空符号，$B\in \Gamma -\sum$，除了输入以外的其他纸带部分初始都为空。
  • $F$是终止状态集合。

图灵机的即时描述和移动

在初始时刻，图灵机接受一个有限的输入字符串，字符串两边是无限的空符号。图灵机处于初始状态并且指向最左边的输入符号。

即时描述(Instantaneous Description, ID)

图灵机的即时描述是一个字符串$\alpha q \beta$

  • 其中$\alpha\beta$是纸带上从最左边的非空符号到最右边的非空符号的部分。
  • 状态$q$放置在正在扫描的符号的左侧。
    • $\alpha\beta q$表示正在扫描最右侧的$B$，可以省略不写。
    • $qB\alpha\beta$表示正在扫描最左侧的$B$，不可以省略不写。

移动(move) $\vdash $

即$ID_1 \vdash ID_2$，根据转移函数进行变化。

如果可以经过0次或多次移动到达新ID，表示为$ID_1 \vdash^* ID_2$

图灵机的语言

图灵机对应的语言可以通过根据终止状态的定义或停机这两种方式来定义。

根据终止状态

$L(M)=\{\omega | q_0\omega \vdash^* I\text{, where I is an ID with a final state} \}$

通过停机

$H(M)=\{\omega | q_0\omega \vdash^* I\text{, and there is no move possible from ID I} \}$

两种表达的等价性

能够构造出等价转换的图灵机。

终止状态到停机

如果$L=L(M)$，则存在另一个图灵机$M’$，使得$L=H(M’)$

  1. 对于$M$的每个终止状态，删除任意移动，使得$M’$在这个状态必然停机。
  2. 令所有没有定义转移函数的非终止状态都定义为向右移动。

停机到终止状态

如果$L=H(M)$，则存在另一个图灵机$M’’$，使得$L=L(M’’)$

  1. 引入新状态$f$，成为唯一的终止状态，且没有移动。
  2. 对于任何没有定义转移函数的二元组，都定义为转换到状态$f$。

递归可枚举语言和递归语言

我们称由图灵机定义的语言类被称为递归可枚举语言(recursively enumerable language) 。

定义算法是一个图灵机，这个图灵机通过终止状态接收，并且无论接受与否，它都注定会停机。

如果$L=L(M)$且图灵机$M$是一个算法，则称$L$是一个递归语言(recursive language)。

每个上下文无关语言都是一个递归语言。

图灵机编程技巧

多道

可以将纸带符号看成一个有多个分量的向量。

标记

增加一个通道来标记特殊位置，在新增加的通道上，大多数符号都是空的，这让图灵机能很方便找到这些位置。

缓存

状态也可以是一个向量，首分量用来控制状态，其余分量用来存储一些信息。

半无穷纸带

可以构造一个图灵机从来不会移动到初始读写头位置的左边。

令初始时读写头的位置为0，右侧位置为1、2、……，左侧为-1，-2，……。

构造一个有两个分道的图灵机，并让增加一个状态分量表示是正在扫描顶部磁道还是底部磁道。

更多限制

扩展的图灵机在本质上表达能力没有变强，只是表示更方便了。

多纸带图灵机有多个纸带，每个纸带有自己的读写头，图灵机的移动取决于它的状态以及每个纸带上读写头下的符号。

用单纸带图灵机模拟多纸带(k)图灵机：

用有2k个分道的单纸带图灵机来模拟，其中对应多纸带图灵机的每个纸带，单纸带图灵机用一个分道来标记读写头，另一个分道记录纸带。

非确定性图灵机

非确定性图灵机(Non-determinstic Turing Machine, NTM) 允许在一次移动的时候有不止一次可能的选择，每一个选择都是一个（状态，符号，方向）三元组，这和 确定性图灵机(Determinstic Turing Machine, DTM)中一样。

用多分道图灵机来模拟广度优先搜索即可。

递归可枚举语言和递归语言的闭包性质

  • 递归可枚举语言和递归语言在并集、拼接、星闭包、翻转、交集以及逆同态操作下都是封闭的。
  • 递归语言在差集、补集操作下封闭。
  • 递归可枚举语言在同态操作下是封闭的。

并集操作

  • 递归语言：只需要看在两个算法上能否停下来。
  • 递归可枚举语言：当两个图灵机有一个接受，但是可能不知道什么时候能停机。

交集操作/差集操作

类似于并集。

拼接操作

  • 递归可枚举语言：用两个单带半无穷图灵机构造一个非确定性图灵机，如果存在一个正确分割方式，那么一定能选择正确的方式。
  • 递归语言：遍历寻找分割方式。

星闭包

和拼接类似。

翻转

只需要用一个辅助图灵机翻转字符串。

逆同态操作

用一个图灵机模拟同态函数$h$，将其应用于输入$\omega$，得到$h(\omega)$，然后用原本的图灵机来接收$h(\omega)$就可以了。

同态操作

  • 递归可枚举语言：设计非确定性图灵机
  • 递归语言：可能取值是无穷的，不适用。

Pumping lemma 和 Ogden's lemma

发表于 2024-11-12

引入

我们已经讨论过了如何判断一个字符串是否属于某个上下文无关语言，使用泵引理即可。

现在介绍一个判断上下文无关语言的更强的引理：Ogden’s lemma（中文翻译为鄂登引理）。

定义

如果一个语言$L$是上下文无关语言，那么存在整数$n$，对$\forall z\in L \wedge |z| \ge n$，我们任意选取至少$n$个位置做标记(marked)，并令$z=uvwxy$，总满足：

  1. $vwx$含有最多$n$个标记
  2. $vx$含有至少一个标记
  3. $\forall i\ge 0,uv^iwx^iy\in L$

区别

显而易见的，泵引理就是Ogden’s lemma标记字符串所有的位置时的情况，由此可见，Ogden’s lemma更加灵活，比泵引理更强，能再泵引理不足以判断的时候起到作用。下面会举个例子。

关于我们用泵引理和鄂登引理到底来干什么，下面的说明会帮助产生一个清晰的认识。

  1. 如果一个语言是上下文无关语言，那么属于这个语言的字符串一定满足泵引理
  2. 如果一个语言不是上下文无关语言，那么属于这个语言的字符串仍然可能满足泵引理
  3. 如果一个字符串满足泵引理，那么这个字符串代表的语言可能是上下文无关语言
  4. 如果一个字符串不满足泵引理，那么这个字符串代表的语言一定不是上下文无关语言

这说明我们一般使用泵引理来判断一个语言不是上下文无关语言（即证明里面该语言存在字符串不满足泵引理）。

使用泵引理说明一个语言是上下文无关语言没有意义。

例

该例子参考stackoverflow的一个问答

考虑语言$L = \{a^ib^jc^kd^l | i=0 \text{ or }j=k=l\}$，证明其不是上下文无关语言。

我们先考虑使用泵引理。

令$z=uvwxy$，假设$z$包含a，我们可以令$v=a,x=\epsilon,y=\epsilon$；假设$z$不包含a，我们可以令$v=b,x=\epsilon,y=\epsilon$。都满足泵引理，因此无法通过泵引理来证明不是上下文无关语言。

现在考虑用鄂登引理。

令$z=ab^nc^nd^n$，标记所有的b，令$z=uvwxy$。

假如v或x有超过一种（区分于一个）字符，则重复就会出现错误的顺序。

所以v和x至少包含一个字符$b$，则出现以下五种情况：

  1. $v=\epsilon,x=b^i$
  2. $v=a,x=b^i$
  3. $v=b^i,x=c^j$
  4. $v=b^i,x=d^j$
  5. $v=b^i,x=\epsilon$
    容易验证以上五种情况都不满足鄂登引理。因此，$L$不是上下文无关语言。
